# ADR 0004: MCP Server

## Status

Accepted

## Context

Mock Machine должен быть удобен не только для ручной работы через админ-панель, но и для автоматизации. Отдельный сценарий - LLM-клиенты, которые могут анализировать неизвестные запросы, создавать моки, переключать сценарии и готовить тестовые состояния без ручного заполнения форм.

## Decision

Добавить поддержку MCP server как официального интерфейса для LLM-клиентов.

MCP server должен предоставлять инструменты для основных операций Mock Machine:

- чтение, поиск, создание, обновление и удаление маршрутов;
- чтение дерева маршрутов;
- управление сценариями ответа;
- переключение активного сценария;
- чтение unknown requests;
- создание маршрута из unknown request;
- чтение health/status сервиса.

MCP server должен использовать те же доменные операции, что и административное API `/mockadminapi`. Web-панель, REST API и MCP не должны реализовывать разную бизнес-логику.

## Rationale

MCP-интерфейс позволяет LLM-агентам работать с Mock Machine напрямую: находить незамоканные запросы, создавать маршруты, переключать ошибки и готовить окружение для разработки или тестов.

Общая доменная логика для панели, API и MCP снижает риск расхождения поведения и упрощает поддержку.

## Consequences

- Все функции панели должны быть доступны через `/mockadminapi`.
- MCP server не должен обходить валидацию и доменные правила backend.
- В будущих версиях авторизация должна быть добавлена согласованно для web-панели, `/mockadminapi` и MCP server.
